# 模型收费文档

## 1. 统一计费规则

- 官方价格：以各模型官方人民币价格为基准。
- 平台售价：`平台售价(元) = 官方价格(元) x 1.2`
- 积分价格：`积分 = 四舍五入(平台售价(元) x 100)`
- 等价写法：`积分 = 四舍五入(官方价格(元) x 120)`
- 汇率：`1 元 = 100 积分`
- 说明：平台仍然只按官方价格的 `1.2` 倍收取，不额外叠加其他加价。

## 2. 程序落地原则

- 图片模型：先算单张价格，再乘 `image_count`，最后统一换算积分。
- 视频模型：先按分辨率、时长、音频、参考素材类型等条件算出官方总价，再统一乘 `120` 换算积分。
- 文本模型：
  - Gemini：按次计费，积分结果只要存在小数部分就直接向上取整。
  - DeepSeek：优先按接口返回的 `usage` 真实 token 数计费。
- 动态计费模型不能对每一秒、每一张、每一段 token 单独取整，必须先算出总官方价格，再统一换算成积分。

#### 4.3.2 token 估算公式

- `token = (宽 x 高 x 24 x 时长秒数) / 1024`
- 程序固定按 `24fps` 估算。
- 480P：`854 x 480`
- 720P：`1280 x 720`
- 1080P：`1920 x 1080`

## 5. 文本模型

### 5.1 Gemini

| 模型 | 计费方式 | 官方价格 | 平台售价 | 积分标准 |
| ---- | ---- | ---- | ---- | ---- |
| gemini-3-pro-preview | 按次 | `0.04元/次` | `0.048元/次` | `4.8 向上取整为 5积分/次` |
| gemini-3-flash-preview | 按次 | `0.03元/次` | `0.036元/次` | `3.6 向上取整为 4积分/次` |

说明：

- Gemini 是当前唯一不走普通“四舍五入”的文本模型。
- 只要积分结果存在小数部分，就直接向上取整到整数积分。

### 5.2 DeepSeek

| 计费项 | 官方价格 | 平台售价 | 积分标准 |
| ---- | ---- | ---- | ---- |
| 输入 token（缓存命中） | `0.20元/百万 token` | `0.24元/百万 token` | `四舍五入(总官方价格 x 120)` |
| 输入 token（缓存未命中） | `2.00元/百万 token` | `2.40元/百万 token` | `四舍五入(总官方价格 x 120)` |
| 输出 token | `3.00元/百万 token` | `3.60元/百万 token` | `四舍五入(总官方价格 x 120)` |

程序公式：

```text
总官方价格(元)
= (缓存命中输入 token / 1,000,000 x 0.2)
+ (缓存未命中输入 token / 1,000,000 x 2)
+ (输出 token / 1,000,000 x 3)

积分 = 四舍五入(总官方价格 x 120)
```

程序读取优先级：

- 以接口返回中的 `usage` 为准。
- 当前已兼容的常见字段包括：
  - `prompt_tokens`
  - `completion_tokens`
  - `prompt_cache_hit_tokens`
  - `prompt_cache_miss_tokens`
  - `prompt_tokens_details.cached_tokens`
- 如果没有拿到可用 `usage`，当前程序按 `0 积分` 兜底，不再回退旧的固定积分。

## 6. 当前程序中的实现约定

- 图片节点：`DoubaoImageCreator`
- 视频节点：`DoubaoVideoGenerator`
- 文本节点：`TextCreation`
- 已接入公式计费的模型范围：
  - 图片：Wan、Seedream、Kling Image、Nano Banana
  - 视频：Wan、Kling、Seedance、Vidu
  - 文本：Gemini、DeepSeek
- 暂未接入公式计费的模型，仍走后台已有静态积分规则兜底；补齐官方价格后再切换到公式计费。
