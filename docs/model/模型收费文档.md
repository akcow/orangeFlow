# 模型收费文档

## 1. 积分换算总公式

- 成本价：以本文档中的人民币成本价为准。
- 销售价：`销售价(元) = 成本价(元) x 1.2`
- 积分价格：`积分 = 四舍五入(销售价(元) x 10)`
- 等价写法：`积分 = 四舍五入(成本价(元) x 12)`
- 汇率：`1 元 = 10 积分`

## 2. 落地规则

- 图片模型：按实际生成张数计费，公式中的单价再乘 `image_count`。
- 视频模型：按实际时长、分辨率、音频、参考素材类型等参数先计算人民币总成本，再统一乘以 `12` 后四舍五入。
- 文本模型：
  - Gemini：按次计费，积分结果只要有小数就直接进 1。
  - DeepSeek：优先按接口返回的 `usage` 真实 token 数计费。
- 动态计费模型不要对每一秒、每一张、每一段 token 分别取整，必须先算出总人民币成本，再统一四舍五入成积分。
- 特殊规则：仅 `Gemini` 文本模型不走四舍五入，计算出积分后只要存在小数部分，就直接向上取整到 `1` 个或更高整数积分。

#### 4.2.2 token 估算公式

- `输出token = (宽 x 高 x 帧率 x 时长) / 1024`
- 程序采用 `24fps`。
- 480p：`854 x 480`
- 720p：`1280 x 720`
- 1080p：`1920 x 1080`

### 5.2 DeepSeek

| 计费项 | 成本价 | 销售价 | 积分标准 |
| ---- | ---- | ---- | ---- |
| 输入 token（缓存命中） | `0.2元/百万token` | `0.24元/百万token` | `四舍五入(总成本 x 12)` |
| 输入 token（缓存未命中） | `2元/百万token` | `2.4元/百万token` | `四舍五入(总成本 x 12)` |
| 输出 token | `3元/百万token` | `3.6元/百万token` | `四舍五入(总成本 x 12)` |

#### 5.2.1 DeepSeek 程序计费公式

`总成本(元) = (缓存命中输入token / 1,000,000 x 0.2) + (缓存未命中输入token / 1,000,000 x 2) + (输出token / 1,000,000 x 3)`

`积分 = 四舍五入(总成本 x 12)`

#### 5.2.2 DeepSeek 返回值使用说明

- 以接口返回中的 `usage` 为准。
- 支持读取以下常见字段：
  - `prompt_tokens`
  - `completion_tokens`
  - `prompt_cache_hit_tokens`
  - `prompt_cache_miss_tokens`
  - `prompt_tokens_details.cached_tokens`
- 如果没有拿到 `usage`，程序当前会按 `0积分` 兜底，不再回退到旧的静态固定积分。
