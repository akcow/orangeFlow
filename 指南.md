# Seedance 模型 Video Generation API 使用教程
Seedance 模型具备出色的语义理解能力，可根据用户输入的文本、图片等内容，快速生成优质的视频片段。通过本教程，你可学习到如何调用 Video Generation API 生成视频。

## 说明
方舟平台的新用户？获取 API Key 及开通模型等准备工作，请参见 [快速入门]。

## 效果预览
### 有声视频生成
仅 Seedance 1.5 pro 支持，输出音画一体的高质量视频，支持环境音、动作音、合成音、乐器音、背景音乐及人声等。
- 输入：一辆地铁轰隆隆驶过，书页和女孩的头发飞扬，镜头开始环绕着女孩360度旋转，周围的背景从地铁站渐渐转变为一个中世纪的教堂，西式幻想风格的音乐渐入。夹在女孩书中的几页信纸随风飞扬，在女孩的周身打着旋，随风而动的纸张降落时，女孩身处的环境已经彻底变成中世纪教堂的模样
- 输入：镜头围绕人物推镜头拉近，特写人物面部，她正在用京剧唱腔唱“月移花影，疑是玉人来”，唱词充满情感，唱腔充满传统京剧特有的韵味与技巧，完美体现了花旦角色的内心世界

### 多参考图生视频
支持上传多张参考图片，模型将依据这些图片的特征和风格，生成与之匹配的动态视频画面。
- 输入：[图1]戴着眼镜穿着蓝色T恤的男生和[图2]的柯基小狗，坐在[图3]的草坪上，视频卡通风格

### 首尾帧生视频
根据输入的首尾关键帧，智能生成过渡画面，形成连贯流畅的视频内容。
- 输入：360度环绕运镜

## 快速开始
视频生成是一个异步过程：
1. 成功调用 `POST /contents/generations/tasks` 接口后，API 将返回一个任务 ID。
2. 你可以轮询 `GET /contents/generations/tasks/{id}` 接口，直到任务状态变为 `succeeded`；或者使用 Webhook 自动接收视频生成任务的状态变化。
3. 任务完成后，你可在 `content.video_url` 字段处，下载最终生成的 MP4 文件。

## 创建视频生成任务
通过 `POST /contents/generations/tasks` 创建视频生成任务。

```python
import os
from volcenginesdkarkruntime import Ark

# Get API Key：https://console.volcengine.com/ark/region:ark+cn-beijing/apikey
client = Ark(api_key=os.environ.get("ARK_API_KEY"))

if __name__ == "__main__":
    print("----- create request -----")
    resp = client.content_generation.tasks.create(
        model="doubao-seedance-1-0-pro-250528", # Replace with Model ID
        content=[
            {
                "text": (
                    "女孩抱着狐狸，女孩睁开眼，温柔地看向镜头，狐狸友善地抱着，镜头缓缓拉出，女孩的头发被风吹动  --ratio adaptive  --dur 5"
                ),
                "type": "text"
            },
            {
                "image_url": {
                    "url": (
                        "https://ark-project.tos-cn-beijing.volces.com/doc_image/i2v_foxrgirl.png"
                    )
                },
                "type": "image_url"
            }
        ]
    )

    print(resp)
```

请求成功后，系统将返回一个任务 ID：
```python
{
  "id": "cgt-2025******-****"
}
```

## 查询视频生成任务
利用创建视频生成任务时返回的 ID，查询视频生成任务的详细状态与结果。此接口会返回任务的当前状态（如 `queued`、`running`、`succeeded` 等）以及生成的视频相关信息（如视频下载链接、分辨率、时长等）。

> 说明：因模型、API负载和视频输出规格的不同，视频生成的过程可能耗时较长。为高效管理这一过程，你可以通过轮询 API 接口来请求状态更新，或通过 Webhook 通知接收通知。

```python
import os
from volcenginesdkarkruntime import Ark

client = Ark(api_key=os.environ.get("ARK_API_KEY"))

if __name__ == "__main__":
    resp = client.content_generation.tasks.get(
        task_id="cgt-2025****",
    )
    print(resp)
```

## 基础使用
### 文生视频
根据用户输入的提示词生成视频，结果具有较大的随机性，可用于激发创作灵感。

```python
import os
import time  
# Install SDK:  pip install 'volcengine-python-sdk[ark]'
from volcenginesdkarkruntime import Ark 

# 初始化客户端
client = Ark(
    base_url="https://ark.cn-beijing.volces.com/api/v3",
    api_key=os.environ.get("ARK_API_KEY"),
)

if __name__ == "__main__":
    print("----- create request -----")
    create_result = client.content_generation.tasks.create(
        model="doubao-seedance-1-0-pro-250528", # 替换为实际Model ID
        content=[
            {
                "type": "text",
                "text": "写实风格，晴朗的蓝天之下，一大片白色的雏菊花田，镜头逐渐拉近，最终定格在一朵雏菊花的特写上，花瓣上有几颗晶莹的露珠"
            }
        ]
    )
    print(create_result)

    # 轮询查询任务状态
    print("----- polling task status -----")
    task_id = create_result.id
    while True:
        get_result = client.content_generation.tasks.get(task_id=task_id)
        status = get_result.status
        if status == "succeeded":
            print("----- task succeeded -----")
            print(get_result)
            break
        elif status == "failed":
            print("----- task failed -----")
            print(f"Error: {get_result.error}")
            break
        else:
            print(f"Current status: {status}, Retrying after 10 seconds...")
            time.sleep(10)
```

## 提示词建议
1. 提示词结构：主体 + 运动，背景 + 运动，镜头 + 运动...
2. 用简洁准确的自然语言描述效果，抽象描述替换为具象描述，重要内容前置。
3. 图生视频建议上传高清图片，图片质量直接影响视频效果。
4. 文生视频随机性较大，可先使用生图模型生成参考图，再做图生视频。
5. 更多技巧参考：Seedance-1.0-pro&pro-fast 提示词指南、Seedance-1.0-lite 提示词指南。

## 进阶使用
### 使用 Webhook 通知
通过 `callback_url` 参数指定回调地址，任务状态变化时方舟会主动推送通知（请求体与查询任务API返回体一致）。以下是接收通知的 Flask 服务示例：

```python
# 基于Flask搭建简易Web Server处理Webhook通知
from flask import Flask, request, jsonify
import sqlite3
import logging
from datetime import datetime
import os

# 基础配置
app = Flask(__name__)
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(levelname)s - %(message)s',
    handlers=[logging.FileHandler('webhook.log'), logging.StreamHandler()]
)
DB_PATH = 'video_tasks.db'

# 初始化数据库
def init_db():
    conn = sqlite3.connect(DB_PATH)
    cursor = conn.cursor()
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS video_generation_tasks (
        task_id TEXT PRIMARY KEY,
        model TEXT NOT NULL,
        status TEXT NOT NULL,
        created_at INTEGER NOT NULL,
        updated_at INTEGER NOT NULL,
        service_tier TEXT NOT NULL,
        execution_expires_after INTEGER NOT NULL,
        last_callback_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
    ''')
    conn.commit()
    conn.close()
    logging.info("数据库初始化完成")

# 核心回调接口
@app.route('/webhook/callback', methods=['POST'])
def video_task_callback():
    try:
        # 解析回调数据
        callback_data = request.get_json()
        if not callback_data:
            logging.error("回调数据为空或非JSON格式")
            return jsonify({"code": 400, "msg": "Invalid JSON data"}), 400

        # 校验必填字段
        required_fields = ['id', 'model', 'status', 'created_at', 'updated_at', 'service_tier', 'execution_expires_after']
        for field in required_fields:
            if field not in callback_data:
                logging.error(f"缺失必填字段：{field}")
                return jsonify({"code": 400, "msg": f"Missing field: {field}"}), 400

        # 提取关键信息
        task_id = callback_data['id']
        status = callback_data['status']
        logging.info(f"收到任务回调 | Task ID: {task_id} | Status: {status}")

        # 写入数据库
        conn = sqlite3.connect(DB_PATH)
        cursor = conn.cursor()
        cursor.execute('''
        INSERT OR REPLACE INTO video_generation_tasks (
            task_id, model, status, created_at, updated_at, service_tier, execution_expires_after
        ) VALUES (?, ?, ?, ?, ?, ?, ?)
        ''', (
            task_id, callback_data['model'], status,
            callback_data['created_at'], callback_data['updated_at'],
            callback_data['service_tier'], callback_data['execution_expires_after']
        ))
        conn.commit()
        conn.close()

        return jsonify({"code": 200, "msg": "Callback received successfully", "task_id": task_id}), 200

    except Exception as e:
        logging.error(f"回调处理失败：{str(e)}", exc_info=True)
        return jsonify({"code": 200, "msg": "Callback received successfully (internal error)"}), 200

# 辅助查询接口
@app.route('/tasks/<task_id>', methods=['GET'])
def get_task_status(task_id):
    conn = sqlite3.connect(DB_PATH)
    cursor = conn.cursor()
    cursor.execute('SELECT * FROM video_generation_tasks WHERE task_id = ?', (task_id,))
    task = cursor.fetchone()
    conn.close()
    if not task:
        return jsonify({"code": 404, "msg": "Task not found"}), 404
    fields = ['task_id', 'model', 'status', 'created_at', 'updated_at', 'service_tier', 'execution_expires_after', 'last_callback_at']
    return jsonify({"code": 200, "data": dict(zip(fields, task))}), 200

if __name__ == '__main__':
    init_db()
    app.run(host='0.0.0.0', port=8080, debug=False)
```

## 使用限制
### 保存时间
任务数据（任务状态、视频URL等）仅保留24小时，超时自动清除，需及时保存生成的视频。

### 限流说明
- **default（在线推理）**：
  - RPM限流：账号下同模型每分钟创建任务数上限，超限制会报错；
  - 并发数限制：账号下同模型同一时刻处理中的任务数上限，超限任务进入队列。
- **flex（离线推理）**：
  - TPD限流：账号单日对同一模型的总调用token上限，超限请求被拒绝。

### 图片裁剪规则
图生视频场景中，若视频宽高比与上传图片不一致，模型会**居中裁剪**图片，规则如下：
1. 计算原始图片宽高比 `Ratio_原始 = 宽/高`，目标宽高比 `Ratio_目标 = A/B`（如16:9对应16/9）；
2. 若 `Ratio_原始 < Ratio_目标`（图片偏竖）：以宽度为基准，垂直居中裁剪高度；
3. 若 `Ratio_原始 > Ratio_目标`（图片偏横）：以高度为基准，水平居中裁剪宽度；
4. 建议上传图片的宽高比与目标视频宽高比尽可能接近，提升视频效果。

### 总结
1. Seedance视频生成为异步流程，核心步骤是「创建任务获取ID → 轮询/通过Webhook查询状态 → 任务成功后下载视频」；
2. 不同模型适配不同场景：1.5 pro侧重音画同步，1.0 pro/fast侧重速度成本，1.0 lite侧重多参考图生成；
3. 关键优化技巧：合理设置输出规格、优化提示词、离线推理降成本、连续视频用尾帧衔接。
